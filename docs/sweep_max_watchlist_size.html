Parameter sweep: max_watchlist_size

Unlike the three optimizer-knob sweeps, this one re-fires the curator at each cap value because the cap shapes the curator's decisions, not just the optimizer's response. Each curve is a separate curator-driven walk-forward through the 5y window, with starter watchlist [AAPL, MSFT, GOOGL, NVDA, SPY] and only the max_watchlist_size input changing. cap=8 is the project default (Sharpe 1.18, the best risk-adjusted result here).

Sharpe = (annualized return − 4% risk-free) / annualized daily-return σ × √252.
Calmar = annualized return / |max drawdown|; penalizes deep drawdowns the way Sharpe doesn't.

A walk-forward check splits the 5y backtest at its midpoint and asks whether the same parameter value wins on each half independently. If yes, the choice is robust across regimes; if no, the full-window winner is at least partly fit to one half. For this sweep (split 2023-09-27): cap=5 wins H1 by both Sharpe and Calmar (H1 Calmar 1.22 vs 0.77 for cap=8); cap=8 wins H2 decisively (H2 Sharpe 1.99 vs 1.32). The H1 result reflects the early-window watchlist's thematic narrowness; cap=8 carries the full 5y window cleanly.
